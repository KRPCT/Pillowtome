<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView UI → SQLite (annotation-store) | User note text + book-derived text_exact/pre/post cross into stored SQL |
| Local device → future P7 sync | change_log rows become the merge ledger; a forged/corrupt clock breaks reconcile |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | annotation INSERT/UPDATE with note/text_* | mitigate | `$n` parameterized binds only — never string interpolation of user/book text (copies locator-store T-02-sql discipline) |
| T-05-02 | Tampering | change_log logical_clock ordering | mitigate | Clock = COALESCE(MAX(logical_clock))+1 computed inside a single atomic INSERT; SQLite serializes writers → monotonic per device, cannot duplicate/reverse |
| T-05-03 | Spoofing | device_id provenance | accept | device_id is locally generated (crypto.randomUUID), never derived from book content; single-user local app, no remote actor in this phase |
| T-05-04 | Info Disclosure | book bytes over IPC | mitigate | Annotations are small structured rows over SQL only; book bytes stay on `pillow://` (D-06). No bytes cross this store |
| T-05-SC | Tampering | npm/cargo installs | accept | No new dependencies this phase (RESEARCH confirmed: foliate vendored, opencc-js/vitest already installed, SHA-256/UUID WebView-built-in). No install step to audit |
</threat_model>
