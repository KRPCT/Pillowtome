<objective>
Wire the two-mode selection + drawing mechanics: turn a text selection into a range-CFI and draw highlights/underlines in BOTH reader surfaces, then restore and replay a work's annotations on open and across mode switches. Paginate goes through foliate's events (`load` / `draw-annotation` / `create-overlayer` / `show-annotation`) — the only path into the closed shadow root. Scroll draws with the CSS Custom Highlight API (live Range, zero manual reflow redraw), per-iframe, lazily as sections (re)load, with a foliate `Overlayer` fallback for WebView < 105.

Purpose: This is the ANNO-01 mechanism — without it there is nothing to highlight and nothing to restore. It also carries the ANNO-04 replay so annotations stay anchored after resize / font change / mode switch via the shared resolver.
Output: `css-highlight.ts` + test; selection + draw + replay wiring in `FoliateView.tsx` (paginate) and `ContinuousScrollStream.tsx` (scroll). The bubble/sheet UI is plan 05-04 — this plan exposes the selection/draw seams it consumes.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted book DOM (section iframe) → highlight draw | Selection ranges + injected ::highlight CSS operate over book content |
| Annotation color/type → ::highlight() CSS registry name | A registry/CSS name built from user input would be a style-injection sink |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering / Elevation | ::highlight() CSS name from annotation color | mitigate | `highlightCssName` builds names only from the fixed cinnabar\|ochre\|green\|indigo allowlist; unknown colors are refused — no arbitrary CSS/registry names |
| T-05-08 | Tampering | drawing into closed shadow via DOM injection | mitigate | Paginate draws only via foliate `draw-annotation`/`addAnnotation`; no shadow querySelector — respects the closed-shadow boundary and avoids brittle DOM injection sinks |
| T-05-09 | Denial of Service | bulk-drawing all annotations on open | mitigate | Lazy per-section draw only (Pitfall 9); perf validated in plan 05-05 stress gate |
| T-05-SC | Tampering | npm installs | accept | No new dependencies (CSS Custom Highlight + foliate Overlayer both in-repo/built-in) |
</threat_model>
