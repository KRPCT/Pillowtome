    在包 `com.pillowtome.app` 新建 `SuppressSelectionActionModeFrameLayout`（继承 `android.widget.FrameLayout`），覆写 `startActionModeForChild(originalView, callback)` 及带 `type` 的重载 `startActionModeForChild(originalView, callback, type)`：当 `type == ActionMode.TYPE_FLOATING`（即文本选择工具条）时返回 `null` 抑制；其余情况 `return super.startActionModeForChild(...)`，保证 app 主 ActionMode 等不被误伤。在 `MainActivity` 覆写 TauriActivity 的 `onWebViewCreate(webView)`（在 generated/ 之外，`tauri android build`/`dev` 不覆盖，只有 `tauri android init` 会重刷）：取 webView 当前 parent，若为 ViewGroup 则记录其在父中的 index 与 LayoutParams，把 webView 从原父 detach，new 一个 `SuppressSelectionActionModeFrameLayout`，用同一 LayoutParams 把 webView addView 进去，再把该 FrameLayout 按原 index/LayoutParams 装回原父。**不要**在容器里消费 window insets（保持 `enableEdgeToEdge` 语义；insets 继续透传给 WebView），否则会破坏 edge-to-edge。onCreate 里的 `enableEdgeToEdge()` 保留不动。

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView 原生宿主 ↔ Tauri window/insets | reparent WebView 进自定义 FrameLayout 跨越 Tauri 的 edge-to-edge/inset 布局边界，可能扰动系统栏与触摸派发 |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-16 | Tampering | reparent 干扰 edge-to-edge / window insets / 触摸派发 | mitigate | 容器不消费 insets、透传给 WebView；Task 3 设备门显式验 insets + 手指竖滑；破裂即按记录回滚 reparent |
| T-05-17 | Repudiation | 原生修复落在 gitignored 且 `tauri android init` 会重刷的目录，可能悄然丢失 | mitigate | `git add -f` force-track 两个 .kt + `.gitignore` 注释点名 + docs 记录 `git checkout` re-apply（Task 1/2） |
| T-05-18 | Denial of Service | 过度抑制误伤 app 主 ActionMode（非文本选择） | mitigate | 仅当 `type == TYPE_FLOATING` 抑制，其余走 `super.startActionModeForChild` |
| T-05-SC | Tampering | npm/cargo installs | accept | 零新依赖，仅新增本地 Kotlin + 文档 |
</threat_model>
