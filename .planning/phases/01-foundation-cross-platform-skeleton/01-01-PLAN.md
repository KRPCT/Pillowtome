<objective>
Stand up the cross-platform Tauri v2 skeleton that every later feature runs inside, and wire the one boundary rule that matters: large book bytes reach the WebView through a Range-aware custom protocol, never over IPC (D-06). This plan scaffolds the app, refactors it into a Cargo workspace with a portable `pillowtome-core` crate plus the `src-tauri` glue crate, vendors foliate-js at a pinned commit, pins every dependency exactly with committed lockfiles (supply-chain baseline), pre-declares all `core` and `src-tauri` module stubs so downstream plans fill files without touching shared roots, and registers the `pillow://` asynchronous URI-scheme protocol with CSP whitelisting.

Purpose: Establishes the shell + byte-streaming boundary that FND-01/FND-02 reading and FND-03 import depend on. Getting the workspace/protocol/IPC boundary right on day 1 is exactly what prevents the Lithium late-refactor trap.
Output: A compiling cross-platform workspace, a registered Range-capable `pillow://` protocol, vendored pinned foliate-js, and empty-but-declared seam modules ready for Plans 02/03 to fill.
</objective>

    In `src-tauri/src/protocol.rs` implement the `pillow` asynchronous URI-scheme handler registered via `register_asynchronous_uri_scheme_protocol("pillow", ...)` on the Builder in `lib.rs`. The handler must: parse the requested id from `request.uri()`; resolve it through a `SourceRegistry` held in Tauri managed state (define the registry in `src-tauri/src/storage.rs` as a `Mutex<HashMap<String, PathBuf>>` for now, with a helper to register/resolve — Plan 05 extends it to `BookSource`); read the `range` request header and respond `200` (full body), `206 Partial Content` with `Content-Range: bytes s-e/len` and `Content-Length`, or `416` when unsatisfiable, capping each served range at 1 MB per the official streaming example.     Enforce V12/V5 scope (01-RESEARCH.md Security Domain): the protocol resolves ONLY ids present in the registry to their registered bytes — never an arbitrary caller-supplied filesystem path. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView (foliate-js/React) → src-tauri protocol | Untrusted book bytes + fetch requests cross into the native protocol handler |
| Third-party packages → build | npm/cargo dependencies + vendored foliate-js enter the trusted build at install time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | pillow:// protocol handler | mitigate | Resolve only registry-known ids; normalize + reject `../` traversal; confine reads to the scoped library/resource dir (never an arbitrary path from the request) |
| T-01-02 | Elevation of Privilege | WebView render context / CSP | mitigate | Strict CSP in tauri.conf.json; scheme whitelisted only where needed; no remote origins in connect-src; book-content JS hardening flagged for Phase 2 |
| T-01-03 | Tampering | Book bytes over IPC (D-06) | mitigate | Bytes stream via custom protocol only; no IPC command carries book bytes — verified by grep gate |
| T-01-SC | Tampering | npm + cargo installs, vendored foliate-js | mitigate | Exact-version pins, committed pnpm-lock.yaml + Cargo.lock, foliate-js vendored at a pinned SHA with retained MIT LICENSE; audit table in 01-RESEARCH.md dispositions all P1 packages Approved (the [SUS] tauri-plugin-android-fs is deferred to and gated in Plan 05) |
</threat_model>
