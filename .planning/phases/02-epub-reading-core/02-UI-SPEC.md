# Phase 2 — UI Design Contract

> Visual and interaction contract for EPUB Reading Core (READ-01..07).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Tone: **极简纸感** (minimal paper-feel) — low-contrast chrome; reading surface is primary.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | radix-nova / neutral / Geist / lucide (code `b2fA`) |
| Component library | radix-ui (via shadcn) |
| Icon library | lucide-react |
| Shell font | Geist Variable — chrome / UI only |
| Reading body font | System CJK stack by default (see Typography); custom fonts via READ-06 |
| CSS variables | true (`src/index.css`) |
| Style base | radix-nova |
| Base color | neutral |

**Already present:**
- `components.json` (style: radix-nova, baseColor: neutral, iconLibrary: lucide)
- `src/components/ui/button.tsx`
- `src/lib/utils.ts` (`cn()`)
- `@fontsource-variable/geist` + Tailwind v4 + shadcn CSS variables in `src/index.css`

**Dual-layer theming rule:**
- **Home shell** (Phase 1 surface, until Phase 4 library theming): keep shadcn `:root` / `.dark` driven by OS preference.
- **Reader root**: apply `data-theme="day|night|sepia"` and override paper-feel tokens under that attribute. Reader theme is independent of OS dark mode.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, slider track padding |
| md | 16px | Default element spacing, sheet section padding-x |
| lg | 24px | Section padding, default page margins |
| xl | 32px | Layout gaps between major sheet sections |
| 2xl | 48px | Toolbar height, major section breaks |
| 3xl | 64px | Page-level spacing (home only) |

**Exceptions (this phase):**
- Touch targets ≥ **44×44px** for all toolbar icon buttons and edge-adjacent controls
- Reader toolbar height: **48px** fixed
- Bottom sheet drag handle hit area: **24px** tall
- Progress bar thickness: **2px** (not on the spacing scale; intentional hairline)
- Immersive tap zones use full viewport height; center strip is **34%** width, left/right **33%** each

## Typography

### Chrome / UI only (Geist Variable)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption / meta | 12px | 400 | 1.4 | Progress %, search result chapter, helper text |
| Body / label | 14px | 400 | 1.5 | Sheet labels, TOC items, search snippets, toolbar book title |
| Control title | 16px | 600 | 1.4 | Section titles inside sheets (阅读模式, 主题, 字体…) |
| Sheet heading | 18px | 600 | 1.3 | Sheet/drawer titles (显示设置, 目录, 搜索) |

Weights declared: **400** and **600** only.

### Reading body (foliate-js renderer CSS — NOT shell)

Applied by injecting styles into foliate render documents; never use Geist for body text.

| Property | Default | Range | Step |
|----------|---------|-------|------|
| Font size | 18px | 12–32px | 1 |
| Line-height | 1.75 | 1.2–2.4 | 0.05 |
| Margins | 24px | 8–48px | 4 |
| Font family | system CJK stack (below) | user-selected + custom imports | — |

**Default font stack (READ-02 default; Phase 3 owns bundled CJK font — do not invent one here):**
```
system-ui, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Noto Sans CJK SC", sans-serif
```

## Layout Contract

### Reader root structure

```
┌─────────────────────────────────────────┐
│ [toolbar 48px]  ← hidden in immersive   │
│ ── progress 2px ─────────────────────── │
│                                         │
│         foliate-view (flex 1)           │
│     + invisible tap-zone overlay        │
│                                         │
│ [bottom sheet]  ← settings / TOC / search│
└─────────────────────────────────────────┘
```

- Root: `position: fixed; inset: 0; display: flex; flex-direction: column`
- Root attribute: `data-theme="day|night|sepia"`
- View host: `flex: 1 1 auto; min-height: 0` (required so foliate-js has explicit height — Phase 1 lesson)
- `foliate-view`: `display: block; width: 100%; height: 100%`

### Top toolbar (48px)

Order left → right:

| Slot | Control | Icon (lucide) | Label / a11y | Action |
|------|---------|---------------|--------------|--------|
| L1 | Back | `ChevronLeft` | 返回 | `onClose` → home shell |
| L2 | Book title | — | truncate 1 line, max ~40% width | non-interactive |
| R1 | TOC | `List` | 目录 | open TOC sheet/drawer |
| R2 | Search | `Search` | 搜索 | open Search sheet |
| R3 | Settings | `Type` (or text **Aa**) | 显示设置 | open Settings bottom sheet |
| R4 | More (optional) | `MoreHorizontal` | 更多 | deferred if empty; do not invent menus |

Progress display (pick one, implement both tokens):
- **Preferred:** 2px progress bar flush under toolbar (`accent` fill, secondary track)
- **Also show:** caption percent in toolbar center-right when chrome visible (`进度 37%`)

Toolbar buttons: ghost/icon variant, 44×44 min hit target, no filled accent backgrounds.

### Immersive mode (READ-04)

| Rule | Spec |
|------|------|
| Default | Immersive ON when reading status becomes `"reading"` |
| Chrome visibility | Toolbar + progress bar hidden; sheets still overlay when opened |
| Show chrome | Tap center strip (34% width, full height) |
| Hide chrome | Tap center strip again, or auto-hide after **3s** idle while reading (optional; if implemented, cancel timer on any sheet open) |
| Paginated left zone | Left 33% width → `renderer.prev()` |
| Paginated right zone | Right 33% width → `renderer.next()` |
| Scroll mode zones | Left/right/center all primarily **toggle chrome**; native scroll remains primary navigation; do not hijack scroll gestures |
| System bars | Prefer fullscreen-friendly layout; do not fight OS status bar on Android beyond safe-area insets |
| Safe area | Toolbar and sheets respect `env(safe-area-inset-*)` |

Tap-zone overlay:
- Transparent absolute layer over `reader__view` when chrome is hidden (and optionally always for page-turn in paginated mode)
- Must not block text selection once annotations land (Phase 5); Phase 2: no selection UI required
- Pointer events: zones receive taps; do not capture multi-touch pinch (leave to WebView/foliate if any)

### Bottom sheet — 显示设置 (Aa)

Entry: toolbar **Aa** / Type button. Same pattern on phone and desktop (bottom sheet, not modal dialog). One-handed phone use is the design driver.

| Property | Value |
|----------|-------|
| Component | shadcn `Sheet` side=`bottom` |
| Max height | 70vh |
| Handle | 24px hit area, centered 32×4px pill |
| Scroll | Internal `ScrollArea` if content exceeds |
| Sections (top → bottom) | see below |

**Sections:**

1. **阅读模式** (READ-01) — `ToggleGroup` single: `分页` | `滚动`. Real-time apply to foliate renderer.
2. **主题** (READ-03) — `ToggleGroup` single: `日间` | `夜间` | `Sepia`. Updates `data-theme` + injects page colors.
3. **字体** (READ-02 / READ-06) — list: system default + imported custom fonts; selected row uses accent indicator. Button `导入字体` below list.
4. **字号** — `Slider` 12–32 step 1; live preview on page; show current value caption (e.g. `18`).
5. **行距** — `Slider` 1.2–2.4 step 0.05; show current value.
6. **边距** — `Slider` 8–48 step 4; show current value.
7. **导入字体** actions — primary text button; helper caption under it.

Section spacing: `xl` (32px) between sections; `md` (16px) within. Separators optional via `Separator`.

### TOC sheet (READ-05)

| Property | Value |
|----------|-------|
| Mobile | Bottom `Sheet` full-height (≤90vh) OR left `Sheet` drawer — **prefer left drawer on width ≥768px, bottom sheet on phone** |
| Title | 目录 |
| List | Nested headings; indent by level (`md` per depth) |
| Item height | min 44px touch row |
| Active item | accent text + left 2px accent bar |
| Action | tap → `foliate goTo` / equivalent → close sheet → remain immersive or briefly show chrome |
| Empty | see Copywriting |

### Search sheet (READ-07)

| Property | Value |
|----------|-------|
| Presentation | Same sheet pattern as TOC (top field sticky) |
| Title | 搜索 |
| Field | shadcn `Input`, autofocus, placeholder `搜索书中内容` |
| Matching | CJK substring — no word-boundary / no space-delimiter requirement |
| Results | `ScrollArea` list: snippet (body 14) + chapter caption (12 muted) |
| Active / selected | accent |
| Tap result | jump via foliate location → close sheet |
| Empty | see Copywriting |
| Debounce | 200–300ms after input idle before search |

### States

| State | Visual |
|-------|--------|
| `loading` | Full-bleed reader surface with muted caption `加载中…` centered (or keep Phase 1 toolbar progress text). No spinner required if caption is clear. |
| `reading` | Immersive by default; foliate content visible |
| `error` | Reuse Phase 1 `ErrorCard` pattern (`src/reader/error-card.tsx`): icon + message + dismiss 返回 |
| Empty search | Centered empty copy in results area |
| Empty TOC | Centered empty copy |
| Font import success | Toast or inline caption under font list: `已导入「{name}」` — keep minimal; no toast system required if inline is clearer |
| Font unsupported | Inline error caption: `仅支持 TTF、OTF、WOFF` |
| Soft-fail malformed EPUB | Existing ErrorCard: `无法打开这本书` / `文件已损坏或无法读取` |

## Interaction Contract

### Requirement → UI mapping

| Req | Interaction |
|-----|-------------|
| READ-01 | Settings sheet → 阅读模式 toggle; apply immediately without reload |
| READ-02 | Settings sheet sliders + font list; live apply to renderer CSS |
| READ-03 | Settings sheet theme toggle; set `data-theme` + inject page bg/fg |
| READ-04 | Default immersive; center tap toggles chrome; L/R thirds page-turn in paginated mode |
| READ-05 | Toolbar 目录 → nested list → goTo |
| READ-06 | 导入字体 → system file picker (TTF/OTF/WOFF) → register face → selectable in 字体 list; 移除 with confirm |
| READ-07 | Toolbar 搜索 → substring query → results → jump |

### Gesture priority (paginated)

1. Open sheet / toolbar button
2. Left/right third page turn
3. Center third chrome toggle

### Gesture priority (scroll)

1. Vertical pan/scroll (native)
2. Tap (any zone) chrome toggle
3. Toolbar / sheet controls

### Persistence (Phase 2 scope note)

- Theme, mode, font size/line-height/margins, selected font: **persist per device for the session at minimum**; prefer local store so reopen keeps prefs. Cross-device sync is Phase 7 — do not design sync UI.
- Reading position restore is Phase 5 (ANNO-04) — show live progress only.

### Accessibility (minimum)

- All icon buttons have `aria-label` in 简体中文
- Focus ring uses accent token inside sheets
- Touch targets ≥ 44px
- Do not rely on color alone for selected toggle — selected = accent fill/outline + weight 600 label

## Token Mapping Notes (implementation hints)

Map reader chrome under `[data-theme="day"]` etc. on `.reader` root, e.g.:

```css
.reader[data-theme="day"] {
  --reader-bg: #F7F3EB;
  --reader-chrome: #EFE8DB;
  --reader-accent: #8B6914;
  --reader-fg: #1C1915;
  --reader-muted: #6B6458;
  --reader-border: #E0D6C4;
  --page-bg: #FFFEF9;
  --page-fg: #1C1915;
}
/* night / sepia analogous */
```

Inject `--page-bg` / `--page-fg` (+ font/size/line-height/margin) into foliate render documents. Shell home continues to use shadcn `--background` / `--foreground`.
