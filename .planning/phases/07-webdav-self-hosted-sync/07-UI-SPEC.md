# Phase 7 — UI Design Contract

> Visual and interaction contract for WebDAV Self-Hosted Sync (SYNC-01..05).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Tone: **极简纸感 + 朱砂点睛** — inherit the Phase 2/3/5 visual language verbatim. Sync UI is quiet chrome: a toolbar button with a status dot, one settings sheet, placeholder cards that look like books, and exactly one user-initiated dialog (进度撤回). Failures are toasts, **never modals** (D-93).

**Scope note:** The sync engine (transport/reconcile/merge/scheduler) is Rust-core plumbing with no UI surface. This spec covers the five user-visible surfaces: **① 书库同步按钮+状态点 · ② 同步设置 sheet · ③ 云端占位卡+点击下载 · ④ 按书「同步此书」开关 · ⑤ 开书合并的痕迹+撤回弹窗**. Triggers themselves (开书拉/合书推, D-90) are silent; only their *visible effects* are specified here.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized) + MUI 7 MD3 theme (already a dependency — see dual-system note) |
| Preset | radix-nova / neutral / lucide (code `b2fA`) — unchanged |
| Component library | radix-ui (via shadcn) for reader chrome; **@mui/material for library-shell chrome** (AppBar, Drawer sheets, Dialog, Snackbar, Menu) — the shipped reality since Phase 4 |
| Icon library | lucide-react (no new icon library) |
| Shell font | Geist Variable — chrome / UI only (MUI theme `CJK_STACK` puts Geist first, then system CJK) |
| Reading body font | foliate render CSS (unchanged; Phase 2/3 own it) |
| CSS variables | true (`src/index.css`) — `--reader-*` aliased to MD3 `--md-*` roles under `.reader[data-theme]` |
| `components.json` registries | `{}` (empty — official shadcn only) |

**Dual-system rule (shipped reality — do not "fix" it this phase):**
- **MUI-rendered chrome** (AppBar buttons, Drawer sheet controls, Dialog, Snackbar, card Menu) takes the MUI theme from `src/theme/mui.ts` — 墨青 ink-teal primary.
- **Custom-CSS chrome** (library cards, chips, reader toolbar, pills, status dots) takes `--reader-*` / `--md-*` tokens — 朱砂 cinnabar primary.
- Phase 7 surfaces use whichever renderer their host surface already uses: sync settings sheet = MUI Drawer (墨青 controls); status dot / placeholder card / trace pill = custom CSS (朱砂 tokens). **Do not introduce a third accent.**

**Reuse, do not reinvent (verified in code):**
- Toast channel: MUI `Snackbar` in `src/App.tsx` (`status` state, bottom-center, `autoHideDuration={4000}`) — the D-93 failure surface. No toast library added.
- Confirm dialog: MUI `Dialog`/`DialogTitle`/`DialogContentText`/`DialogActions` pattern (取消 / 删除) in `src/App.tsx`.
- Undo pill: `.reader__undo-pill` in `src/reader/ReaderBottomBar.tsx` (RotateCcw + 返回原位) — the exact visual pattern for the D-92 sync trace pill.
- Long-press card menu: MUI `Menu`/`MenuItem` in `src/library/LibraryCard.tsx` (500ms long-press / right-click, currently holds 删除) — D-98's toggle extends this menu.
- Settings sheet: `LibrarySettingsSheet` → `src/reader/SettingsSheet.tsx` (MUI `Drawer` bottom, sections with `Divider`) — the 同步 section extends it; the sync config form is a new sibling sheet with the same Drawer pattern.
- Sheet scroll structure contract (CLAUDE.md touch/scroll gate #3): header `shrink-0`, body `flex-1 min-h-0 overflow-y-auto`, `touch-action: pan-y`, `-webkit-overflow-scrolling: touch`. **The sync settings sheet MUST honor this** and be finger-scroll verified on AVD.
- Existing shadcn ui: `button, input, popover, scroll-area, separator, sheet, slider, switch, textarea, toggle-group, toggle`. Existing MUI usage: `AppBar, Box, Button, Chip, Dialog, Divider, Drawer, IconButton, List(ListItemButton/ListItemText), Menu/MenuItem, Paper, Slider, Snackbar, Stack, Switch, ToggleButton(Group), Toolbar, Tooltip, Typography`.

## Spacing Scale

Inherits Phase 2 scale (multiples of 4): xs 4 · sm 8 · md 16 · lg 24 · xl 32 · 2xl 48 · 3xl 64.

**Exceptions (this phase):**

| Element | Value | Rationale |
|---------|-------|-----------|
| All interactive touch targets | ≥ **44×44px** | Global touch gate (toolbar sync button, switches, menu rows, dialog actions) |
| Sync status dot | **8px** circle | Unobtrusive persistent state (D-93); sits at the sync IconButton's top-right |
| Status dot offset | **2px** from icon corner (exception, not on scale) | Optical alignment on a 24px icon inside a 44px hit area |
| 占位卡 cloud badge | reuse `.library-card__badge` metrics (top/right 6px, 4px radius) | Same slot as the existing 已读 badge — one badge slot, never both at once |
| Card download progress bar | **3px** height | Reuses `.library-card__progress` metrics exactly (accent fill on dark track) |
| Form field vertical gap in sync sheet | **16px** (md) | MUI `Stack spacing={2}` precedent in SettingsSheet |
| Sync sheet section gap | **32px** (xl) between sections, `Divider` between | SettingsSheet precedent |
| Sync settings sheet max height | `min(85vh, 720px)` (new exception) | Form content (3 fields + 2 switches + path + status) exceeds the 70vh reading-prefs sheet; still Drawer-bottom, body scrolls per touch gate |
| Trace pill (D-92) | reuse `.reader__undo-pill` metrics verbatim | One pill language in the reader |
| Dialog (撤回/断开) | MUI `Dialog` defaults (24px padding, 12px radius) | Existing delete-confirm precedent |

### C. Status dot semantics (D-93) — no new hue invented

| Sync state | Dot |
|------------|-----|
| 未配置 | **无点**；按钮图标用 muted |
| 空闲 / 上次成功 | **无点**；图标正常 fg |

5. trace pill 的「已从其他设备同步」文案前的小图标（CloudDownload, accent 色）——pill 本体为 surface 底 + fg 字，**不**整体 accent

**Explicitly NOT accent:** 同步设置 sheet 里的主按钮（MUI → 墨青）、撤回弹窗按钮（MUI → 墨青）、失败状态点（error）、「未同步」灰态（muted at 55% opacity）。

## Layout Contract

### 1. 书库同步按钮 + 状态点 (D-90 manual · D-93 status)

Location: `App.tsx` AppBar `Toolbar`, inserted **immediately left of the settings gear** (rightmost cluster: 示例 · 导入 · 扫描 · **同步** · 设置).

| Rule | Spec |
|------|------|
| Component | custom `SyncStatusButton` wrapping MUI `IconButton` (44×44 hit) |
| Icon | lucide `RefreshCw`（同步中可旋转 1 圈/s；reduced-motion 时静态） |
| Status dot | absolutely-positioned 8px circle, top-right, 2px inset; colors per §C |
| Tap behavior | 未配置 → 打开 `SyncSettingsSheet`；已配置 → 触发手动同步（D-90 兜底）；同步中 → tap 忽略（aria-busy） |
| a11y | `aria-label="立即同步"`；失败时 `aria-label="立即同步，上次同步失败"` |
| Long-press | 无（菜单不属于工具栏按钮；配置入口在设置 sheet） |

### 2. 同步设置 sheet `SyncSettingsSheet` (SYNC-01 · D-95/D-96/D-97/D-104)

Entry points: (a) `LibrarySettingsSheet` 新增「同步」分节（见 §6）；(b) 未配置时点书库同步按钮。Presentation: MUI `Drawer` `anchor="bottom"`, same shell as `SettingsSheet` — rounded top, drag handle, max-height `min(85vh, 720px)`, header `shrink-0` + scrollable body（touch gate #3）.

Sections (top → bottom):

1. **服务器** — MUI `TextField` ×3: 服务器地址（placeholder `https://dav.jianguoyun.com/dav`，坚果云一等公民示例）、用户名、应用密码（`type="password"`）。Helper caption under 应用密码: `建议使用应用密码，不要使用账户主密码`（D-96）。
2. **安全** — 两个独立 MUI `Switch` 行（D-95，默认均 OFF）：
   - `允许 HTTP（明文，仅局域网）` — ON 时下方出 warning caption（error 色 caption，12px）：`明文 HTTP 会暴露你的凭据，请仅在可信局域网内使用`
   - `信任自签名证书` — ON 时 warning caption：`仅在确认证书来自你自己的服务器时开启`
3. **远端路径** — `TextField`，默认 `pillowtome/`。Helper caption（D-104 约束明示）：`多台设备必须填写相同路径`
4. **设备名称** — `TextField`（可编辑；默认取设备型号/主机名）。Helper：`同步提示里显示的本机名字`（CONTEXT discretion 项 — 本契约采纳「可编辑设备名」，它喂给 D-92 撤回弹窗文案）
5. **操作行** — 主按钮 `测试并保存`（MUI contained, 墨青）＋次按钮 `取消`。测试中按钮 loading 态 `正在测试连接…`；结果用 inline caption 呈现（成功 = accent/正常色 `连接成功，已保存`；失败 = error 色，四类分类文案见 Copywriting，D-97）。**测试未通过不得保存**。
6. **同步状态**（仅已配置时显示，D-93 详情）— 只读行：服务器（URL 截断）、上次同步时间（相对时间，如 `3 分钟前`）、状态（`正常` / `同步失败：{原因}`）。行末次按钮 `立即同步`。
7. **断开连接**（仅已配置时显示）— destructive text 行。确认走 MUI Dialog（见 Copywriting）。仅移除本机配置与 keychain 凭据；远端数据保留。

Password field rule: 密码只进 keychain，**绝不**回显进状态详情、toast、日志或 IPC 结构体（研究 Pattern 4）。重新打开 sheet 时密码框为空 + caption `凭据已保存在系统密钥环`。

### 3. 云端占位卡 + 点击下载 (SYNC-04 · D-99/D-102 · research Q3)

`LibraryCard` 的状态变体（同一组件族，不是新卡片）：书目经状态平面到达、文件不在本机时渲染。

```
┌─────────────┐
│  ┌───────┐  │   cover 区 = 现有字母 placeholder（封面字节不随状态平面走，
│  │  书   │ ☁ │          D-100 单一真相：下载后经 import-pipeline 重解析）
│  │       │  │   ☁ = cloud badge（lucide CloudDownload, 复用 已读 badge 槽位）
│  └───────┘  │
│  ▓▓▓░░░░░░  │   下载中：3px accent 进度条（复用卡片进度条通道）
│ 书名         │
│ 作者         │
└─────────────┘
```

| State | Visual | Interaction |
|-------|--------|-------------|
| 可下载（对端开了文件同步） | placeholder + cloud badge；无 dim | tap → 后台开始下载，卡片转「下载中」；**不**立即开书 |
| 下载中 | 3px accent 进度条 + caption `下载中 {n}%` | tap 忽略（research Q3：禁用打开）；下载完成 → import-pipeline 重解析 → 变正常卡 |
| 下载失败 | 卡片回到「可下载」+ toast（D-93 软失败） | 再次 tap 重试 |
| 未同步（对端未开文件同步，D-102 灰态） | 整卡 `opacity: 0.55`，badge 换 lucide `CloudOff` + 文字 badge `未同步` | tap → Snackbar `该书未开启文件同步，进度与批注仍会同步`；不可打开 |
| 文件已在本机 | 正常 LibraryCard（现有渲染，含封面/进度） | 现有行为 |

进度/批注对所有书生效、不受文件平面限制（D-102）——占位卡的阅读进度条照常显示（若有进度）。

### 4. 按书「同步此书」开关 (SYNC-04 · D-98)

Location: 现有 `LibraryCard` 长按/右键 MUI `Menu`（删除 之上插入）。**无全局「自动上传全部」开关**（D-98）。

| Rule | Spec |
|------|------|
| Menu item | `同步此书`，前置 lucide `Check`（仅已开启时显示）或 `CloudUpload` 图标 |
| Behavior | tap → 切换该本 `file_sync.enabled`，关 Menu；toast 反馈（见 Copywriting） |
| Upload 进行中 | 菜单项 caption 变 `正在上传…`，tap 忽略 |
| 占位卡（对端书） | 菜单项不渲染（开关属于持有文件的一端） |

### 5. 开书合并：痕迹 + 撤回弹窗 (D-92 · SYNC-05 的唯一冲突可视化)

Trigger: 开书拉取对账后，对端进度更远 → **静默跳到最远**（不打断阅读、无 modal），并留痕迹。

| Element | Spec |
|---------|------|
| Trace pill | 复用 `.reader__undo-pill` 样式与位置（ReaderBottomBar 区域，进度条旁）。内容：CloudDownload 小图标（accent）+ `已从其他设备同步`。区别于手动跳转的「返回原位」pill——两者同一槽位，sync pill 优先 |
| 存续期 | 本会话内，直到：(a) 点击 → 弹窗；(b) 用户手动跳转（scrub/TOC/翻页 pill 被新跳转替换）；(c) 合书。撤回窗口 = 会话内（research Pitfall 7：本地原位暂存内存） |
| 弹窗（点击 pill） | MUI `Dialog`——**用户主动发起**，不受 D-93「失败不弹模态」约束。标题 `已从其他设备同步`；正文 `「{设备名称}」上读到了 {n}%，已自动跳到最远位置。`；actions：`撤回原位`（MUI 墨青主按钮）/ `保留进度`（inherit 次按钮） |
| 撤回 | 跳回本地原位 + pill 消失。原位由 merge 副产物携带（merge_progress 输出，research Pattern 2）——UI 不自己猜位置 |
| 无更远进度 | 无 pill、无 toast——静默（与 ANNO-04 的 silent-restore 一致） |

**批注/书目合并 = 无 UI**。唯一例外（research Pattern 2 非破坏兜底）：同 id 同 revision 不同内容的批注副本落库时 note 前缀 `冲突副本`——批注 sheet 原样渲染该前缀即可，不加任何专门 UI（CONTEXT：可视化冲突管理已 deferred）。

### 6. LibrarySettingsSheet 新增「同步」分节

插在现有分节之后（字体/排版之后、`Divider` 分隔）：

| State | Content |
|-------|---------|
| 未配置 | 标题 `同步` + body `未开启同步` + caption `配置 WebDAV 服务器后，阅读进度与批注可在多设备间同步` + CTA `设置 WebDAV 同步` → 打开 `SyncSettingsSheet` |
| 已配置 | 标题 `同步` + 只读行 `{服务器 URL}` · `上次同步 {相对时间}`（失败时 error caption `同步失败：{原因}`）+ 行 `同步设置` → 打开 `SyncSettingsSheet` |

### States 汇总

| State | Visual |
|-------|--------|
| 手动同步成功 | 无 toast（状态点归于无点即反馈）；可选 `同步完成` Snackbar——**采纳可选**，与书库现有 status 频道习惯一致 |
| 同步失败（任何触发源） | Snackbar `同步失败：{原因}`，bottom-center，4s 自动消失，**绝不弹 modal**（D-93） |
| 首次配置成功 | sheet 内 inline `连接成功，已保存`；不自动触发全量上传（文件同步按书开启，D-98） |
| 开书拉到更远进度 | 静默跳 + trace pill（§5） |
| keyring 不可用（Linux 无 Secret Service 等） | sheet 内 error caption `系统密钥环不可用，无法保存凭据`；禁用 测试并保存 |
| 服务器限流（坚果云 503/429） | toast `同步失败：服务器限流，请稍后重试`；引擎指数退避，UI 不做倒计时 |

## Interaction Contract

### Requirement → UI mapping

| Req | Interaction |
|-----|-------------|
| SYNC-01 | 设置 sheet「同步」分节 / 书库同步按钮 → `SyncSettingsSheet` → 填服务器+应用密码 → `测试并保存`（D-97 强制）→ 凭据入 keychain（密码不过 IPC，前端只见 `{configured, url, username}`） |
| SYNC-02 | 开书自动拉（无 UI）；对端更远 → §5 trace pill + 撤回弹窗；手动 → 书库同步按钮 |
| SYNC-03 | 与 SYNC-02 同通道（批注合并完全静默，无专门 UI） |
| SYNC-04 | 长按书卡 → `同步此书`；对端 → §3 占位卡点击下载；灰态「未同步」卡（D-102） |
| SYNC-05 | 进度撤回弹窗（§5）是 v1 唯一冲突可视化；`冲突副本` 前缀为批注兜底；失败一律 toast + 状态点 + 设置页原因（D-93） |

### Trigger visibility (D-90/D-91)

- 开书拉 / 合书推 / 切后台推：**零 UI**（不打断阅读；合书推失败在下次进书库时由状态点+toast 呈现）。
- 手动同步按钮：唯一的主动触发；同步中状态点脉冲 + 按钮 aria-busy。
- 无后台定时同步（D-91）——**不设计**任何「自动同步间隔」设置项。

### Gesture & touch gates (CLAUDE.md)

- `SyncSettingsSheet` 是 Drawer-bottom 表单 sheet：header `shrink-0`，body `flex-1 min-h-0 overflow-y-auto` + `touch-action: pan-y` + `-webkit-overflow-scrolling: touch`；表单超屏必须手指竖滑可达 测试并保存——**AVD 上验证**，不只桌面滚轮（touch gate #4）。
- 占位卡在现有 `.library-shell__body` 滚动容器内，不加任何覆盖层（touch gate #1）。
- 长按出菜单与现有 删除 共用同一 500ms/10px 手势实现——不为同步开关发明第二个手势。
- ReaderBottomBar 的 trace pill 不扩大可点区域之外的热区；不遮挡 scrubber 拖动。

### Accessibility (minimum)

- 所有 icon 按钮 简体中文 `aria-label`（立即同步 / 同步设置 / 下载此书）。
- 状态点不只靠颜色：失败态同时改按钮 `aria-label`（`上次同步失败`）；同步中 `aria-busy="true"`。
- 「未同步」灰态卡 `aria-disabled="true"` + `aria-label="{书名}，未同步"`。
- Switch 行 label 与控件关联（MUI `FormControlLabel` 模式）；warning caption `role="note"`。
- Touch targets ≥44px；focus ring 用各渲染层自己的 primary（MUI 墨青 / 自定义朱砂）。

### Device acceptance (CLAUDE.md gate)

本阶段 UI 验收必须跑 **AVD 生产 APK**（`pnpm tauri android build --debug --target x86_64 --apk` + force-stop 冷启动），覆盖：sync sheet 手指滚动、状态点三态、占位卡下载流、撤回弹窗、失败 toast。桌面绿 ≠ Android 绿。
