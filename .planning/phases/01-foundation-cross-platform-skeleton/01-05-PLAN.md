<objective>
Deliver FND-03: import a book from device storage through the opaque `BookSource` storage-handle (never a raw path — D-05), and make Android SAF-granted access persist across app restarts. Desktop import resolves a filesystem path to `BookSource::Path`; Android import resolves a `dialog.open` `content://` URI to `BookSource::ContentUri` and — the hard part — persists the OS grant via `takePersistableUriPermission` so a force-stop + relaunch can reopen the book without re-granting. There is NO official Tauri API for persistable URI permission, so this plan explicitly chooses between the community `tauri-plugin-android-fs` (audit + pin) and a small native Kotlin plugin, preferring local auditable code.

Purpose: Android scoped storage makes raw paths meaningless; the storage-handle abstraction plus persisted grants is what lets the library survive restarts on mobile. This is the last foundation piece and the second of the two on-device (non-autonomous) proofs.
Output: `BookSource`-backed import (desktop + Android), grant persistence + re-hydration, a recorded SAF-approach decision, and a human-verified restart-persistence check.

This plan is NON-autonomous: it contains a decision checkpoint (SAF approach + supply-chain audit of a [SUS] community crate) and a human-verify checkpoint requiring a real emulator restart cycle (D-13).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Android SAF grant → durable OS state | A persisted `content://` permission is long-lived OS-registered state re-hydrated on launch |
| Community crate (`tauri-plugin-android-fs`) → build | If chosen, a [SUS] third-party crate + its Kotlin enters the trusted build |
| User-picked file → fs read scope | The imported file/URI is read by the fs plugin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SAF | Information Disclosure / Tampering | SAF persisted grant | mitigate | Persist only the user-picked URI; re-hydrate on launch; fs scope limited to picked file / app library dir, not arbitrary storage |
| T-01-SC | Tampering (supply-chain) | tauri-plugin-android-fs ([SUS]) | mitigate | blocking-human decision + legitimacy audit (crates.io + source review), exact rev/SHA pin, committed Cargo.lock; prefer native Kotlin to avoid the trust surface entirely |
| T-01-11 | Elevation of Privilege | native Kotlin install-time hooks | mitigate | If community plugin chosen, audit Kotlin/Gradle for install-time hooks (treat `postinstall`/`prepare`/`.pth`-equivalents as code execution per CLAUDE.md) before build |
</threat_model>
