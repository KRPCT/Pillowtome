# Android build notes

How to build and run 枕籍 on Android, and the environment traps that cost real time in Phase 1.

> **Why this file exists:** `src-tauri/gen/android/` is generated by `cargo tauri android init` and is
> **gitignored**. Two of the fixes below live inside that directory, so they are lost on regeneration
> and on a fresh clone. Everything needed to re-apply them is here.

## Prerequisites

| Component | Requirement | Notes |
|---|---|---|
| Android SDK | platform + build-tools (35 or 36 verified) | `ANDROID_HOME` must point at it |
| Android NDK | r27 (`27.2.12479018` verified) | `NDK_HOME` must point at it |
| Rust targets | `aarch64-linux-android`, `armv7-linux-androideabi`, `i686-linux-android`, `x86_64-linux-android` | added to the toolchain you build with |
| JDK | **17 or 21** — *not* 25 | see trap 3 |
| Device | physical device, or an AVD | Phase 1 verified on AVD `Medium_Phone_API_36.1` (API 36) per decision **D-13** |

`minSdk` is **26** (decision D-12).

## Run

```bash
# 1) start an emulator (or plug in a device)
"$ANDROID_HOME/emulator/emulator" -avd <avd-name>

# 2) from the repo root
pnpm tauri android dev
```

Do **not** pass `--target` to `android dev` — it takes an optional positional `[DEVICE]`; the ABI is
derived from the connected device. (`android build` *does* accept `--target`.)

## Environment traps

### 1. Windows: symlink creation is denied

`tauri-cli` symlinks the compiled `.so` into `app/src/main/jniLibs/<abi>/`. Windows blocks this unless
one of:

- **Developer Mode** is on (Settings → System → For developers), or
- the terminal runs **as Administrator**.

Symptom: `Creation symbolic link is not allowed for this system.`

Verify with a non-elevated shell:

```powershell
New-Item -ItemType SymbolicLink -Path "$env:TEMP\_lnk" -Target "$env:TEMP" | Out-Null; Remove-Item "$env:TEMP\_lnk"
```

### 2. Gradle distribution download can time out

`gen/android/gradle/wrapper/gradle-wrapper.properties` points at
`services.gradle.org`. If that host is unreachable the build dies with
`SocketTimeoutException: Read timed out` before anything else runs.

If a Gradle distribution of the **same version** is already in the wrapper cache under a different
variant, point the wrapper at it — `-all` is a superset of `-bin`, so it is a drop-in:

```properties
# gen/android/gradle/wrapper/gradle-wrapper.properties
distributionUrl=https\://services.gradle.org/distributions/gradle-8.14.3-all.zip
```

Check the cache first:

```bash
ls ~/.gradle/wrapper/dists/          # look for gradle-<version>-{bin,all}/<hash>/*.ok
```

The `.ok` marker means Gradle already unpacked and accepted that distribution; the wrapper will reuse
it without any network access. The cache directory name is `base36(md5(distributionUrl))`, which is
why the URL must match the variant exactly.

> **Do not** substitute a third-party Gradle mirror. That widens the supply-chain surface for a
> build-time code-execution artifact. Reuse an already-verified local distribution, or fetch from
> `services.gradle.org` and verify the published SHA-256 (`distributionSha256Sum`).

### 3. JDK 25 breaks Gradle 8.14.3

Gradle 8.14.3 supports up to **JDK 24**. If a newer JDK is first on `PATH`, the build fails with
`Unsupported class file major version` (or a JVM-version rejection) *after* the distribution unpacks —
so it looks unrelated to Java.

Two levers, and you generally want both:

```properties
# gen/android/gradle.properties  -> pins the Gradle *daemon* JVM
org.gradle.java.home=/path/to/jdk-21
```

```bash
# pins the gradlew *launcher* JVM
export JAVA_HOME=/path/to/jdk-21
```

`gradle.properties` alone is insufficient — `gradlew` itself starts under `JAVA_HOME`/`PATH`.

### 4. `A problem occurred starting process 'command 'pnpm.bat''` is a red herring

`gen/android/buildSrc/.../BuildTask.kt` tries `pnpm` → `pnpm.exe` → `pnpm.cmd` → `pnpm.bat` and its
`catch` swallows **any** exception — including a process that launched fine but exited non-zero. It
then reports only the *last* candidate's name.

The underlying command is `pnpm tauri android android-studio-script --target <abi>`, which opens a
**WebSocket back to a running `tauri android dev` session** to fetch CLI options. Consequently:

- Running `gradlew assembleX86_64Debug` standalone can **never** succeed — it fails with
  `ConnectionRefused` masked behind `pnpm.bat`.
- Always drive Android builds through `pnpm tauri android dev` / `pnpm tauri android build`.

### 5. Gradle daemons capture their environment at start

`project.exec` inside a Gradle task spawns processes from the **daemon's** environment, not from your
current shell. If a daemon was started with a different `PATH`/`JAVA_HOME`, changing your shell has no
effect. After changing environment, stop the daemons:

```bash
gen/android/gradlew --stop
```

## Device gate (mandatory for agents)

**Desktop green + unit tests green ≠ Android correct.** Phase 1 and Phase 2 both shipped bugs that only
showed on the emulator (CORS, APK resource paths, hand-rolled `pillow://` URLs, Windows symlink
denial, foliate-js paginator centering short pages as a floating card on tall screens).

### When the gate applies

Any change that touches:

- `src/reader/**` (chrome, themes, typography, TOC, search, fonts, immersive)
- `src/components/ui/**` used by sheets / scroll areas
- `src/lib/pillow.ts` or `pillow://` protocol / CSP
- Import / SAF / storage-handle paths
- Anything that ships UI or book bytes to the WebView

**must** complete this checklist before claiming done:

1. AVD `Medium_Phone_API_36.1` running (`adb devices` shows `device`).
2. `pnpm tauri android dev` installs and launches `com.pillowtome.app` (on Windows: **Developer Mode
   ON** or run elevated so `jniLibs` symlinks work — trap 1).
3. Manual or screenshot review of the affected surface (open sample EPUB is the minimum).
4. **If the surface scrolls** (settings sheet, TOC, search results, scroll reading mode): **finger
   pan-y** on the device — desktop mouse wheel is not enough.
5. Note any residual device-only risk in the plan SUMMARY / VERIFICATION.

Skip only pure-Rust `core/` unit-test-only changes with no UI/protocol surface.

### Touch / scroll pitfalls (global)

| Anti-pattern | Symptom on Android | Fix |
|--------------|--------------------|-----|
| Full-screen transparent overlay with `pointer-events: auto` over content | Book / list cannot scroll or swipe | Only capture taps where needed; leave body free for pan-y |
| Radix `ScrollArea` with only `max-h-*` on the root | Sheet content clipped, **no finger scroll** | Parent `flex flex-col max-h-[…]`; body `flex-1 min-h-0 overflow-y-auto` (see `.reader-sheet__body`) |
| Missing `viewport-fit=cover` + safe-area floor | UI under system status bar | `index.html` viewport-fit + CSS `--safe-top: max(28px, env(...))` |

These are **project constraints** (also in `CLAUDE.md`). Do not reintroduce.

```bash

export NDK_HOME="$ANDROID_HOME/ndk/27.2.12479018"
```

## Platform behaviours worth remembering

These are not bugs to fix; they are properties of the platform that the code now accounts for.

- **`bundle.resources` are not filesystem files on Android.** They are packaged inside the APK as
  Android assets (`assets/…`), reachable only through `AssetManager`. `BaseDirectory::Resource` +
  `std::fs::read` works on desktop and fails on Android. The bundled sample EPUB is therefore embedded
  with `include_bytes!` and materialized into `app_data_dir()` on first launch.
- **Custom-protocol URLs differ per platform** and additionally depend on
  `app.security.dangerousUseHttpScheme`. Never hand-roll them from `navigator.userAgent`; call
  `convertFileSrc(id, 'pillow')` from `@tauri-apps/api/core`. Per `tauri/src/app.rs`:
  Windows & Android → `http://<scheme>.localhost/<path>`; macOS, iOS & Linux → `<scheme>://localhost/<path>`.
- **The WebView never shares an origin with a custom protocol**, in dev (Vite dev server) or in release
  (`tauri.localhost`). Custom-protocol responses must carry `Access-Control-Allow-Origin`, or `fetch`
  fails before the response status is observable.
- **foliate-js `margin` is header/footer band height (`px`), not page padding.** Page insets belong in
  `setStyles` body CSS. Default `max-block-size: 1440px` centers short content on tall phones — set
  `max-block-size` to the host height (see `applyFoliateLayoutAttrs`).

## Windows desktop note (not Android-specific)

The default `stable-x86_64-pc-windows-gnu` toolchain on the Phase-1 development machine has a broken
`gcc.exe` (exits 1 with no output on any compile or link). Desktop builds there must use MSVC:

```bat
call "C:\BuildTools\VC\Auxiliary\Build\vcvars64.bat"
set RUSTUP_TOOLCHAIN=stable-x86_64-pc-windows-msvc
```

No `rust-toolchain.toml` is committed — pinning MSVC would break Linux/macOS contributors. Android
cross-compiles use NDK clang and are unaffected.
