<objective>
Land SYNC-01: the user can configure and connect a self-hosted WebDAV server, with credentials stored in the OS keychain and never synced. This plan delivers the Rust-side connect/config spine — keychain credential storage (`credentials.rs`), the `reqwest_dav` transport with TLS/http gating and the D-97 Chinese error classes (`transport.rs`), a wiremock off-device integration matrix (`tests/sync_transport.rs`), the first three IPC commands (`sync_get_config` / `sync_test_and_save` / `sync_disconnect`), and the Android keychain spike (ndk-context init + AVD production-APK smoke) that is the phase's highest-risk integration point.

Purpose: every later sync plan (state plane, file plane, reconcile) builds on a working, tested transport and a credential store that provably survives process death on Android.
Output: the four Rust modules + wiremock suite + the Android smoke checklist. Depends on 07-00 (dependency pins + Cargo.lock commit, SCHEMA_V8 with `sync_config`/`sync_state`/`sync_file_state`, `core/src/sync/` skeleton); this plan consumes those and adds no schema of its own.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView → Rust IPC | `SyncConfigInput` (incl. the password, in plaintext by necessity) crosses IN; only `PublicSyncConfig` (password-free by type) crosses OUT |
| Rust → OS keychain | Password persists only via `keyring` (Windows Credential Manager / macOS Keychain / Secret Service / Android Keystore) — never SQLite, never the WebDAV remote, never logs |
| Rust → WebDAV server | First network egress of the project: Basic-auth PROPFIND/MKCOL/PUT to a user-configured, possibly hostile or broken server |
| Untrusted remote input | PROPFIND multistatus XML and HTTP status/headers from the server are parsed as untrusted input (remote JSON validation lands with the state plane in 07-02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
