## Color

### Chrome (unchanged)

Reuse `--reader-bg / --reader-chrome / --reader-accent / --reader-accent-soft / --reader-fg / --reader-muted / --reader-border` under `.reader[data-theme]`. Do **not** introduce parallel chrome tokens. Note editor / annotations sheet surfaces = `--reader-chrome`; text = `--reader-fg`; secondary text = `--reader-muted`; separators = `--reader-border`; active/selected = `--reader-accent`.

### Annotation palette (D-70: 4 colors) Рђћ NEW tokens

Highlights render as **translucent backgrounds** (CSS Custom Highlight `::highlight(){ background-color }` in scroll; foliate SVG `Overlayer.highlight` with `--overlayer-highlight-opacity` in paginate). Underlines (D-71) use the **solid** seed as `text-decoration-color`. Swatches in the bubble use the solid seed. Each color must stay legible over `--page-bg` in all three themes Рђћ night uses lighter, desaturated tints so page text (`--page-fg`) reads through.

| Palette key | Name | Day / Sepia seed (solid) | Night tint (solid) | Highlight fill |
|-------------|------|--------------------------|--------------------|----------------|
| `cinnabar` | Тю▒уаѓ (default, first swatch) | `#D24A32` | `#E8846F` | seed @ **28%** alpha (day/sepia), **30%** (night) |
| `ochre` | УхГ | `#C08A2E` | `#D9B061` | same alpha rule |
| `green` | ж╗Џу╗┐ | `#4F855F` | `#7FB48C` | same alpha rule |
| `indigo` | жЮЏУЊЮ | `#3E5C99` | `#8AA4D6` | same alpha rule |

**Implementation:** declare per-theme CSS vars under `.reader[data-theme]`, e.g. `--anno-cinnabar: #D24A32; --anno-cinnabar-fill: color-mix(in srgb, #D24A32 28%, transparent);` (night block overrides with the tint + 30%). `::highlight(pillow-hl-cinnabar){ background-color: var(--anno-cinnabar-fill); }` and `::highlight(pillow-ul-cinnabar){ text-decoration: underline; text-decoration-color: var(--anno-cinnabar); }`. Paginate maps the same seed to `Overlayer.highlight`/`underline` `{color}`.

**Accent (`--reader-accent` Тю▒уаѓ) reserved for (unchanged from Phase 2, extended):**
1. Active/selected list item in annotations sheet (left 2px bar + accent text)
2. Selected filter segment (тЁежЃе/жФўС║«/угћУ«░/С╣дуГЙ, if implemented)
3. Focus ring on note textarea and sheet controls
4. Active (bookmarked) state of the toolbar bookmark toggle

Do **not** use `--reader-accent` as a highlight color Рђћ the 4-color palette is a distinct token family. The `cinnabar` highlight seed (`#D24A32`) is intentionally close to but not identical to the accent, so a Тю▒уаѓ highlight and an accented UI element never visually merge.

## Interaction Contract

### Requirement Рєњ UI mapping

| Req | Interaction |
|-----|-------------|
| ANNO-01 | Select text Рєњ bubble Рєњ tap color swatch Рєњ highlight drawn (scroll: CSS Custom Highlight; paginate: foliate `draw-annotation`) |
| ANNO-01 | Bubble СИІтѕњу║┐ Рєњ underline (independent type, D-71) |
| ANNO-02 | Bubble угћУ«░ Рєњ NoteEditorSheet Рєњ textarea Рєњ т«їТѕљ; note attaches to the highlight |
| ANNO-03 | Toolbar bookmark toggle Рєњ point-CFI at current position; appears in annotations sheet |
| ANNO-04 | No direct UI Рђћ reopen/resize/mode-switch/cross-device restore is silent via shared resolver; annotations sheet row-tap jump exercises the same anchor path |

### Gesture priority (must not regress Phase 2 / CLAUDE.md gate)

- **Scroll mode:** native vertical pan is priority #1 Рђћ bubble and highlights add zero pointer-capture surface. Selection uses native long-press handles (D-74). CSS Custom Highlight adds no DOM layer.
- **Paginate mode:** existing L/R page-turn + center chrome zones unchanged; selection handles come from the system inside the (closed-shadow) iframe. Bubble sits above via foliate `load`-provided doc.
- Bubble/sheet dismiss must not swallow the next tap intended for reading.

### Persistence & sync

- Annotations persist per `work_id` in SQLite (schema V7); soft-delete tombstone (D-80). **No sync UI this phase** Рђћ change_log rows are written but not transmitted (Phase 7 owns sync surfaces). Do not design conflict/merge UI.

### Accessibility (minimum)

- All bubble/toolbar icon buttons have у«ђСйЊСИГТќЄ `aria-label` (жФўС║«┬иТю▒уаѓ / СИІтѕњу║┐ / угћУ«░ / тцЇтѕХ / тѕажЎц / Ти╗тіаС╣дуГЙ / уД╗жЎцС╣дуГЙ).
- Color swatches are not color-only: selected swatch shows a ring/check, and each has an `aria-label` naming the color.
- Focus ring uses accent; touch targets РЅЦ44px.
