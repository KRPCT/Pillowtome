<objective>
Deliver FND-04: a pure-`core`, off-device unit-tested DRM/corruption detector that refuses cleanly and never decrypts (D-10). `detect_protection(epub_bytes)` reads the EPUB zip read-only, classifies protection three ways (clean / font-obfuscation-only / content-DRM), refuses Adobe ADEPT + Kindle + unknown-algorithm content encryption, and soft-fails corrupt/truncated zips through a typed error so the app shows a friendly card instead of crashing.

Purpose: Establishes the DRM safety boundary before any book is served for rendering. Because the detector is pure Rust in `pillowtome-core` with no tauri/platform deps, it is fully testable off-device — the fastest feedback loop in the phase and the right place to be exhaustive about the encryption.xml three-case ambiguity (Pitfall 4).
Output: `Protection` classification, `CoreError` typed errors, four tiny EPUB fixtures, and a green `cargo test -p pillowtome-core protection::` suite.

This is a TDD plan: fixtures + failing assertions (RED) precede the implementation (GREEN).
</objective>

  <action>
    Implement `pub fn detect_protection(epub_bytes: &[u8]) -> Result<Protection, CoreError>` and `pub enum Protection { None, FontObfuscationOnly, ContentDrm(&'static str), Unknown }` in `core/src/protection.rs` using the `zip` crate to read the archive READ-ONLY (never decrypt — D-10):

    1. Open the zip from an in-memory cursor; a bad/truncated zip maps to `CoreError::Corrupt` (soft-fail, Pitfall 5). Treat a missing `META-INF/container.xml` as `CoreError::Corrupt` too.
    2. If `META-INF/rights.xml` is present → `Protection::ContentDrm("Adobe ADEPT")` (refuse).
    3. Else read `META-INF/encryption.xml` if present and `classify_encryption(xml)`: if EVERY `EncryptionMethod/@Algorithm` is a known IDPF/Adobe font-obfuscation algorithm applied only to font resources → `Protection::FontObfuscationOnly`; if any resource is encrypted with an unknown/retailer algorithm → `Protection::ContentDrm(...)` (refuse); ambiguous → `Protection::Unknown` (refuse).
    4. No encryption.xml and no rights.xml → `Protection::None`.
    5. Detect Kindle containers by magic bytes (a `.mobi`/`.azw`/KF8 blob is not an EPUB zip) and refuse via `ContentDrm("Kindle")` or `CoreError::Unsupported`.

    Enforce the V5/V12 zip-slip guard from the Security Domain: when iterating archive entries, normalize each entry name and reject/skip any path containing `..` or absolute components — even on the read-only detect path — so a malicious archive cannot influence any later extraction. Keep `detect_protection` allocation-light and side-effect-free.

    The refuse/allow policy for `FontObfuscationOnly` in P1 is: classification is exact, and the render layer (Plan 04) MAY refuse-with-message or render; either satisfies D-10 (per 01-RESEARCH.md Assumption A4). This function only classifies; it does not decide UX.
