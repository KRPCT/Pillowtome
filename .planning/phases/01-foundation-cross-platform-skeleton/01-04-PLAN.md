<objective>
Prove the render pipeline end-to-end on BOTH targets (FND-01 desktop + FND-02 Android): open one bundled DRM-free sample EPUB, have foliate-js paginate and render a readable page, and turn one page — on Windows/macOS/Linux desktop AND on the Android emulator (AVD `Medium_Phone_API_36.1`, the D-13 emulator substitute for real hardware). Book bytes arrive via `fetch(pillow://...)` (D-06), and a book that fails `detect_protection` shows a clean error card instead of rendering.

Purpose: This is the cross-platform reading proof — the milestone that shows the whole shell + protocol + engine chain works identically on both platforms. It is the thin slice only: open + render + one page-turn. Full reading UX (modes/themes/TOC/search) is Phase 2.
Output: The reader slice (`FoliateView.tsx` + error card + "打开示例书籍" wiring), a bundled sample EPUB, Android build config (minSdk 26), and two human-verified render confirmations.

This plan is NON-autonomous: FND-02 requires a running emulator and a human visual confirmation (D-13), and the desktop render check is a manual smoke per 01-VALIDATION.md.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted EPUB → foliate-js render context (WebView) | Book HTML/CSS/JS renders inside the WebView |
| Book bytes → WebView via pillow:// | Bytes stream from disk to foliate-js through the custom protocol |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Elevation of Privilege / Info Disclosure | foliate-js render of untrusted EPUB | mitigate | Strict CSP (Plan 01) blocks remote origins; only the pillow scheme is fetchable; full book-content JS sandboxing/sanitization is scheduled for Phase 2 and noted here |
| T-01-10 | Denial of Service | opening a DRM/corrupt book | mitigate | DRM gate calls `detect_protection` before `view.open`; failure renders the error card, never crashes (D-10) |
| T-01-03 | Tampering | book bytes over IPC (D-06) | mitigate | Bytes reach foliate-js only via `fetch(pillow://...)`; grep gate confirms no IPC byte transfer |
</threat_model>
